# Atelier UI — Vue

A workshop toolkit for hands-on AI development training. 22 LLM-optimized components, zero runtime dependencies, and a built-in MCP server so your AI assistant can look up exact prop names instead of guessing them.

**Package:** `@atelier-ui/vue` &nbsp;·&nbsp; **Elements:** `LlmButton`, `LlmDialog`, … &nbsp;·&nbsp; **Style:** CSS custom properties

---

## Quick start

```bash
npm install @atelier-ui/vue
```

```css
/* main.css or global stylesheet */
@import '@atelier-ui/vue/styles/tokens.css';
```

```ts
// main.ts
import { createApp } from 'vue';
import App from './App.vue';

createApp(App).mount('#app');
```

```vue
<script setup lang="ts">
import { LlmButton } from '@atelier-ui/vue';
</script>

<template>
  <LlmButton variant="primary">Save</LlmButton>
</template>
```

## What's inside

| Category | Components |
|---|---|
| **Inputs** | Button, Input, Textarea, Checkbox, Toggle, Select, RadioGroup |
| **Display** | Badge, Card, Avatar, Skeleton, Progress |
| **Navigation** | Breadcrumbs, Tabs, Pagination, Menu |
| **Overlay** | Dialog, Drawer, Tooltip, Toast |

Two-way binding uses `v-model` and `v-model:value` conventions where applicable. Toast exposes a `useLlmToast` composable.

## MCP server

The MCP server is hosted and pre-configured in workshop workspaces. Add it manually to any project:

```json
{
  "mcpServers": {
    "storybook-vue": {
      "type": "http",
      "url": "https://atelier.pieper.io/storybook-vue/mcp"
    }
  }
}
```

Once connected, your AI assistant can call `get_component_docs`, `search_components`, `get_stories`, and `get_theming_guide` — getting exact element names, prop types, and v-model bindings without hallucinating the API.
