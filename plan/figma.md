## Workflow

### Design Token Change
1. Update value in Figma: `figma_update_variable` with the variable ID
2. Copy the new value to `libs/llm-components-angular/src/styles/tokens.css`
3. Mirror to `libs/llm-components-react/src/styles/tokens.css`
4. Screenshot key Storybook stories to confirm no visual regression

### New Component
1. Create a new Section on the Components page
2. Build frames for all variants/states using `UI Tokens` variables
3. Screenshot with `figma_take_screenshot` — iterate until correct
4. Run generator: `nx generate @atelier-ui/generators:llm-component --name=<name>`
5. Implement Angular + React simultaneously, referencing Figma values
6. Verify parity with Storybook screenshot

### Changing a Component's Visual Design
1. Update the Figma section frame first
2. Screenshot and confirm it looks right
3. Update `.css` file in Angular lib (`:host` selector)
4. Mirror to React lib (`.llm-<name>` selector)
5. Final parity check

## Token → CSS Mapping Quick Reference

When Figma gives you a hex from a variable, map it to CSS like this:

```css
/* Figma: color/primary (Light) = #2563eb */
background-color: var(--ui-color-primary); /* #2563eb */

/* Figma: spacing/4 = 16 */
padding: var(--ui-spacing-4); /* 1rem = 16px */

/* Figma: radius/md = 8 */
border-radius: var(--ui-radius-md); /* 0.5rem = 8px */
```
