# Atelier — Lessons

Patterns and gotchas captured during implementation. When a mistake repeats, add a rule here so it doesn't repeat again.

## Figma plugin API (figma-console MCP)

- **`node.resize(w, h)` flips auto-layout axis sizing to FIXED on both axes.** If you want a frame to auto-size its height (`primaryAxisSizingMode = 'AUTO'`) after setting a width, set the mode *after* calling `.resize()`, not before. Otherwise the frame collapses to the resized height regardless of children. This bit me in Progress body, Combobox panel/wrapper, CodeBlock wrapper/body, and Table inner container — all of which silently clipped or collapsed.
- **Setting `layoutMode` on a frame *after* calling `.resize()` reverts both axes to AUTO.** The frame then shrink-wraps to its children — e.g. a 32×32 circle with a "1" text inside collapses to a narrow pill. After flipping layoutMode, explicitly re-assert `primaryAxisSizingMode` and `counterAxisSizingMode` to `FIXED` and call `.resize()` once more, or set layoutMode first and size last.
- **`primaryAxisAlignItems='SPACE_BETWEEN'` with a single child centers it.** This bit the CodeBlock `no-copy` header (single `typescript` text centered) and every Combobox option row without a checkmark. Use `'MIN'` plus `layoutGrow=1` on the first child to push later items to the end.
- **Moving a SECTION on the page does NOT move its children.** Section children are positioned in page-absolute coordinates. After bumping a section's `.y`, re-apply the same delta to every child (`child.y += deltaY`), or reposition the compSet at `(sec.x + 24, sec.y + 92)` explicitly.
- **Text in a fixed-width container won't wrap unless you set `textAutoResize='HEIGHT'`** — Figma defaults to `WIDTH_AND_HEIGHT` so long strings overflow and get clipped by parent `clipsContent`. Setting HEIGHT then explicitly resizing the text width = content-area width makes it wrap properly.
- **`node.layoutPositioning = 'ABSOLUTE'` requires the node's parent to already have `layoutMode !== 'NONE'`.** Parent the node first (`parent.appendChild(node)`), then set `layoutPositioning`. Setting it on an unparented or NONE-layout child throws.
- **`counterAxisAlignItems` enum is `'MIN' | 'MAX' | 'CENTER' | 'BASELINE'`** — not `'START'`/`'END'`. `START` will throw a validation error.
- **REST-backed Figma tools (screenshots, `figma_get_component_image`) 403 without `FIGMA_ACCESS_TOKEN`.** Use structural inspection via `figma_execute` (read `fills`, `boundVariables`, dimensions) when the REST path is unavailable. The Desktop Bridge channel still works for creation/modification.
- **Node-ids in URLs use `-` instead of `:`** — `parameters.design` links like `figmaNode('420-87')` map to Figma node `420:87`.

## Design-system parity

- **Bind every color in a new Figma design to a UI Tokens variable, never a raw hex** — otherwise Dark mode won't follow. Use `figma.variables.setBoundVariableForPaint(paint, 'color', variable)` and discover variables with `figma.variables.getLocalVariablesAsync()`.
- **Before writing new component CSS, check if an equivalent semantic token already exists** (`--ui-color-on-primary`, `--ui-color-text-on-danger`, etc.). Hard-coding `#fff` or `#000` for "on colored background" bypasses the token layer and breaks theming.
- **Storybook `parameters.design` lives on both the meta and every named story** — meta links to the component set, individual stories link to specific variants. Pattern established by `libs/angular/src/lib/button/llm-button.stories.ts:5-9,38-74`.

## Docs site redesign (2026-05-01 batches)

- **Astro `Icon name={literal}` uses string-literal union autocomplete; data-driven callers (`SECTION_ICONS[key]`, BottomNav `icon` prop) need `IconName | (string & {})` in the prop type or the typecheck explodes** with `ts(2322): Type 'string' is not assignable to type 'IconName'`. Lesson: when extracting a typed icon registry, leave a controlled-escape so consumers driven by data don't need a cast at every call site. `& {}` keeps the literal autocomplete while widening the runtime type to string.
- **`navigator.clipboard.writeText` rejects with `NotAllowedError: Document is not focused` when invoked from devtools / a backgrounded tab.** Programmatic copy-button tests fail in the test surface even when the wiring is correct. Lesson: don't take a backgrounded `await navigator.clipboard.writeText` failure as a real bug; verify by clicking in a focused window. Same caveat applies to programmatic `.focus()` and `:focus-visible` matching — keyboard intent (real Tab key) is the only thing that triggers `:focus-visible`, not `el.focus()`.
- **`@media (prefers-color-scheme: dark)` source swaps via `<picture>` don't react to a JS-set `data-theme` override.** When the topbar theme toggle writes `data-theme="dark"` on `<html>` while the OS prefers light, `<picture>`'s media-source resolution is locked at parse time — images stay light. Lesson: pair the media query with a `MutationObserver` on `<html>`'s `data-theme` attribute and rewrite `<img>.src` from a sibling `data-light-src` cache when the override flips.
- **Astro's scoped `<style>` rewrites class selectors to `.foo[data-astro-cid-XXX]` so cssText scans for bare `.foo:focus-visible` miss the rule.** Verifying focus-visible wiring via DOM stylesheet inspection requires querying the rule by `selectorText` containing the base class, not as a substring of `cssText`. The rule still applies at runtime — just don't trust naïve substring greps.
- **Brand-recurring phrase coverage is case-sensitive.** `rg "single source of truth"` misses sentence-starting `Single source of truth`. When auditing recurring-phrase land-rate, use `rg -i` or check both. Don't double-place a phrase because the first instance is sentence-cased.
- **The Astro dev toolbar + Claude in Chrome dev affordances inject DOM siblings outside `#docs-shell` (`<a id="bottomBar">`, `<astro-dev-toolbar>`).** A site-wide axe-core run flags these as accessibility violations even though they don't ship in production. Lesson: scope axe to `{ include: [['#docs-shell']] }` to filter dev-only injections out of the report — same approach for any future automated a11y CI.
- **Page-hero / section-head / pillar / code-card chrome lives in `landing.jsx` + `landing.css` at `skills/atelier-design/ui_kits/docs-site/`.** Always read those two files when building a new section before writing CSS from scratch — kit specifies exact radius (lg=14), exact stroke (1px), exact ease (`cubic-bezier(0.16, 1, 0.3, 1)`), and exact crosshair-grid mask (`radial-gradient(ellipse 800px 500px at center, #000 30%, transparent 75%)`). Token-driven recreation lands in one batch; freelancing the chrome leaks into a five-batch reconciliation later.
- **Inline `navigator.clipboard.writeText` patterns must keep the success-state visual swap inside the `try` block after `await` resolves.** Earlier index.astro logged `console.error` on failure but still flipped the button to "Copied" — visual state misrepresented reality. Lesson: only flip success after the await; the catch path keeps the button in its original state and writes the failure to an `aria-live` region.
- **`opacity:` reductions silently break dark-mode contrast.** `.docs-nav-link--back { opacity: 0.65 }` on `--ui-color-text-muted` (#94a3b8) sat at 3.78:1 in dark — below WCAG AA 4.5:1. Lesson: any text with `opacity < 0.85` needs an explicit contrast pass against both light + dark surfaces. Prefer dropping opacity in favor of a real muted token rather than blending an opaque color over a dark background.
- **Astro `<style>` blocks need explicit `<style>` import on each consumer; primitives that ship their own scoped `<style>` block have those styles hashed per-instance.** When extracting a primitive (CodeCard, Pillar, CompCard), keep the `<style>` block colocated rather than hoisting to global.css if you want isolation; hoist when the chrome is shared and you want a single source. The MCP card was extracted into CodeCard with isolation; pillars went to global.css for cross-page reuse.
